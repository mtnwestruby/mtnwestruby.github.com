"Zero to Async with EventMachine and Goliath" by Gerred Dillon

Evented programming is all the rage lately, but actually understanding how it works and how to operate with it can lead programmers down a difficult road. This workshop will start from the very basics - how an event loop works, and build up with EventMachine. Goliath, a framework built on the marriage of EventMachine and Fibers, will be introduced. Benefits, concerns, caveats, and new ways to think about programming and interacting with users will all be covered. For the most interested in evented programming, Node.js will also be introduced as an alternative for evented programming.

Gerred Dillon (@justicefries) brings years of experience building evented systems on top of EventMachine and Node.js. He loves building products and APIs with blazingly fast I/O, and helping others realize freedom from threaded programming. He is the co-organizer of Boulder JavaScript, the organizer of Denver JavaScript, and spends almost all of his time working on asynchronous systems and data science.

"Advanced Rails Design: Use Fancy OO Techniques Without Turning Ruby into Java" by Noel Rappin

As Rails applications have grown and become more complex, limitations of the classic "Fat model, skinny controller" architecture have become evident. Large model classes and tight database coupling become difficult to maintain and slow to test. There have been many attempts to manage this issue. Overgeneralizing, these structures tend to add additional architecture layers that use ActiveRecord as a dumb model and use some form of dependency injection to allow for more dynamic management of object relationships.

All this sounds great, but if you came to Rails precisely because it didn't have the kind of architectural overhead that more complex frameworks required then you're probably looking at these ideas with some skepticism.

In this workshop, we'll show how to apply these models to your daily Rails development without burying you in a pile of complicated overhead. We'll talk about dependency injection, role or service based architecture, and how to use these to make your ongoing development life easier. We'll also cover how these techniques make it easier to write low-overhead super-fast tests to make your TDD process fly.

Noel Rappin (@noelrap) is a Senior Engineer in the Engineering Talent Development department at Groupon. His team is responsible for internal training and coaching, recruiting, and external outreach. Before joining Groupon, Noel was a Senior Consultant at Obtiva, where he lead teams on a variety of web development projects for clients.

Noel has authored five technical books including "Rails Test Prescriptions" and the forthcoming "Productive JavaScript" both from the Pragmatic Press, and has taught about web development topics around the country.

"Create your own CI Server status light & siren workshop" by JT Zemp

A DIY, hands-on workshop that involves soldering, electronics assembly and coding (arduino and ruby) with the end result that each person doing the workshop leave with a hardware device that will connect to their CI server and will sound an alarm if the build fails, and give the green light if the build worked. "Writing a Rails Engine" by Erik Michaels-Ober

As of Rails 3.2, plugins are officially deprecated. They have been replaced by Rails Engines, a class of public methods for hooking into Rails applications. This workshop will teach you how to write a Rails engine from scratch or convert your existing plugin to an engine. We will cover writing generators, migrations, configuration, mounting, and packaging your engine as a gem. We will also discuss best practices for making your engine work with other engines and when to factor your existing Rails code into an engine, for reuse across multiple applications.

Erik Michaels-Ober (@sferik) is the author of one of the most popular Rails engines: RailsAdmin. He also helps maintain OmniAuth and a number of other Ruby gems. He was a 2010 Ruby Summer of Code mentor and a 2011 Google Summer of Code mentor, while serving as a Fellow at Code for America. He is currently employed as a software engineer at Yardsale, a startup based in San Francisco, California.

"Design a Cucumber/Selenium2/Rspec Test Suite Against a Source-Ambiguous Web Application that is both Powerful and Stakeholder-Friendly" by Sam Rawlins

We will create a Cucumber/Selenium2/RSpec test suite against an existing, source-unavailable web app. We will quickly go over the Cucumber Scenario (Gherkin) syntax and basic Selenium2 methods. With the foundations laid, we will explore how to use "Background," data tables, "Transform," write stakeholder-friendly scenarios, deal with asynchronous activity, be unambiguous about step names, use Cucumber tags and profiles, handle ambiguously-located DOM elements, highlight elements before interacting with them, and allow for loosely-written scenarios.

A Cucumber/Selenium2 beginner will hit the ground running in this workshop, which will be too much fun. /s?he/ will leave with a solid understanding of the tools, and how to design /his|her/ first Cucumber/Selenium2 test suite. A developer who has experience writing Gherkin scenarios, or experience using Selenium to drive a browser will benefit from the advanced techniques. Selenium2/WebDriver is very different from Selenium 1, and learning Selenium2 from the ground up in a workshop would be very beneficial.

Sam Rawlins (@srawlins) has been coding in Ruby for about 5 years now. He writes Ruby programs and scripts largely in the Systems world, in both work and my hobbying. He maintains several C extensions and supports them cross-platform. This talk stems from a project at his day job, where he maintains a Java web application that is tested partially with regression tests that use Cucumber, Selenium 2, and RSpec.
